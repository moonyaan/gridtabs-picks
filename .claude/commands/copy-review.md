# 营销文案审查

## Purpose

在发布前审查营销材料，确保文案流畅自然、无 AI 痕迹、适合目标受众。

## 适用场景

- 网站文案（首页、产品页、落地页）
- App Store / Google Play 描述
- 社交媒体内容
- 推广邮件
- 任何面向用户的营销文本

## Instructions

当用户提供营销文案时，按以下维度审查：

### 1. 流畅度检查

- 句子是否通顺，逻辑是否连贯
- 是否有语法错误或拼写错误
- 段落之间过渡是否自然

### 2. 自然度检查

- 是否像真人写的，而非机器生成
- 是否有过度正式或生硬的表达
- 是否符合目标语言的表达习惯（中文要像中文，英文要像英文）

### 3. AI 痕迹识别

常见 AI 写作特征：
- 过度使用 "此外"、"总之"、"值得注意的是" 等连接词
- 句式过于工整、对仗
- 空洞的形容词堆砌（"强大"、"卓越"、"无缝"）
- 过于完美的结构（总分总、三段论）
- 缺乏具体细节和个性化表达
- 英文中过度使用 "leverage"、"streamline"、"empower"、"delve" 等

### 4. 场景适配检查

- App Store：是否突出核心价值、是否有明确的行动号召
- 社交媒体：是否有吸引力、是否适合快速阅读
- 邮件：是否有个人感、是否避免垃圾邮件特征词
- 网站：是否清晰传达价值主张

## Output

输出格式：

```

**整体评分**: X/10

**问题汇总**:
- [严重程度] 问题描述

**具体问题**:

### 流畅度
- ...

### 自然度
- ...

### AI 痕迹
- ...

### 场景适配
- ...

## 修改建议

[具体的修改建议，可以是逐句对比或重写版本]

## 优化后版本（可选）

[如果用户需要，提供完整的优化版本]
```

## 自定义选项

用户可以在调用时指定：
- 目标场景（App Store / 邮件 / 社交媒体 / 网站）
- 目标语言（中文 / 英文 / 其他）
- 品牌调性（正式 / 轻松 / 专业 / 友好）
- 是否需要直接输出优化版本

## Example Usage

```
/copy-review
[粘贴文案内容]
```

```
/copy-review App Store 中文 轻松风格
[粘贴文案内容]
```

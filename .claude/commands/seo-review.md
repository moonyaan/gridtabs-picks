# /seo-review - SEO 审查

## Purpose

在 draft pick 发布前检查 SEO 质量，确保内容在搜索引擎中有竞争力。针对 gridtabs picks 的具体场景优化。

## Input

```
/seo-review                          # 审查最新的 draft
/seo-review ai-coding-tools          # 审查指定 slug 的 draft
```

## Instructions

### Step 1: 定位 Draft

- 如果用户指定了 slug，读取 `picks/drafts/{slug}.md`
- 如果没指定，读取 `picks/drafts/` 目录，找最近修改的 `.md` 文件
- 如果 drafts 目录为空，提示用户先用 `/curate` 生成 draft

### Step 2: 解析 Draft

从 markdown draft 中提取：
- slug（frontmatter）
- 英文标题 + 中文标题
- 英文描述 + 中文描述
- 所有网站的 name、URL、英文描述、中文描述

### Step 3: 逐项审查

#### 3.1 Slug 检查

| 规则 | 标准 |
|------|------|
| 长度 | 3-5 个词，用 `-` 连接 |
| 可读性 | 看 slug 就能猜到主题 |
| 无冗余词 | 不要 the、a、and、best 等填充词 |
| 全小写 | 无大写字母 |
| 关键词 | 包含核心主题词 |

#### 3.2 英文标题检查

| 规则 | 标准 |
|------|------|
| 长度 | 30-60 字符（Google 截断约 60） |
| 核心关键词 | 标题前半部分包含主题关键词 |
| 吸引力 | 让人想点击，不是干巴巴的列表名 |
| 无 AI 味 | 不含 ultimate、comprehensive、essential、curated 等 |
| 无堆砌 | 不是关键词罗列 |

#### 3.3 中文标题检查

| 规则 | 标准 |
|------|------|
| 长度 | 10-25 个字符 |
| 自然度 | 像中文母语者会说的话 |
| 非直译 | 不是英文标题的翻译 |
| 有吸引力 | 让中文用户想点开 |

#### 3.4 英文描述检查

| 规则 | 标准 |
|------|------|
| 长度 | 80-155 字符（Google meta description 上限约 155） |
| 关键词 | 自然包含 1-2 个目标关键词 |
| 行动感 | 告诉读者能获得什么，不只是描述内容 |
| 具体 | 有数字或具体信息（如网站数量、适用场景） |
| 独特性 | 不能和其他 pick 的描述雷同 |

#### 3.5 中文描述检查

| 规则 | 标准 |
|------|------|
| 长度 | 30-80 个字符 |
| 自然度 | 说人话 |
| 非直译 | 独立撰写 |

#### 3.6 网站描述检查

对每个网站的英文和中文描述：

| 规则 | 标准 |
|------|------|
| 英文长度 | 40-100 字符 |
| 中文长度 | 15-50 字符 |
| 主题相关 | 描述要扣主题，不是通用介绍 |
| 差异化 | 每个网站描述角度不同，不要模板化 |
| 无重复句式 | 不要每条都 "A tool that..." 开头 |

#### 3.7 整体 SEO 分析

- **目标关键词推断**：根据主题推断用户可能搜索的关键词
- **关键词覆盖**：标题、描述、网站描述中是否自然覆盖这些关键词
- **搜索意图匹配**：内容是否匹配潜在搜索意图（如 "best design tools 2026"）
- **与已发布 picks 的差异**：读取 `picks/picks.json`，确保不和已发布内容重复定位

### Step 4: 输出报告

```
## SEO 审查报告: {slug}

**整体评分**: X/10

### 问题汇总

| 项目 | 状态 | 问题 |
|------|------|------|
| Slug | ✅/⚠️/❌ | ... |
| EN Title | ✅/⚠️/❌ | ... |
| ZH Title | ✅/⚠️/❌ | ... |
| EN Description | ✅/⚠️/❌ | ... |
| ZH Description | ✅/⚠️/❌ | ... |
| Site Descriptions | ✅/⚠️/❌ | ... |
| Keywords | ✅/⚠️/❌ | ... |

### 详细问题

#### {项目名}
- 当前：`{当前内容}`
- 问题：{具体问题}
- 建议：`{优化后的内容}`

### 推荐关键词

| 关键词 | 搜索意图 | 当前覆盖 |
|--------|---------|---------|
| ... | ... | ✅/❌ |

### 优化后版本（如有修改建议）

完整输出优化后的 draft 内容，用户可直接采用。
```

## 评分标准

| 分数 | 含义 |
|------|------|
| 9-10 | 可直接发布 |
| 7-8 | 小问题，微调即可 |
| 5-6 | 需要修改几处 |
| 3-4 | 问题较多，建议重写部分内容 |
| 1-2 | 需要大改 |

## 重要规则

- **给具体建议**——不要只说"标题太长"，要给出缩短后的版本
- **参考已发布内容**——读 picks.json 了解现有风格，保持一致性
- **中英文分开审**——不要用英文 SEO 标准套中文
- **优化后版本保持 draft markdown 格式**——方便用户直接替换
